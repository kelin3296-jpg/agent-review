# Agent Review Prompt

The runtime prompt is generated by `agent_review.adapters.base.build_reviewer_prompt`.
This file documents the contract for packaged installs.

Reviewer agents read only the size-capped `review-brief.md` handoff by default.
If the brief says the diff was truncated, they may inspect the current project
files listed in the brief. Runtime metadata, JSON files, command logs, and prior
reviewer outputs are for tool debugging, not review context.

First extract acceptance criteria from the user's request, system constraints,
host-agent promises, and explicit project rules. Do not invent new requirements.
Use `FIX` only for clear unmet requirements, necessary evidence missing, test or
build failures, safety risks, format errors, or scope risks. Put optional
improvements in `suggestions` while keeping `PASS`.

They return JSON only:

```json
{
  "status": "PASS",
  "summary": "Short conclusion.",
  "severity": "none",
  "acceptance_checklist": [
    {
      "id": "AC-1",
      "criterion": "Acceptance criterion.",
      "source": "User request",
      "result": "pass",
      "evidence": "Evidence."
    }
  ],
  "issues": [],
  "fix_instructions": [],
  "suggestions": [],
  "can_deliver": true,
  "reviewer_confidence": "medium"
}
```
